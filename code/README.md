# GPU Merge-Path Project

This project implements the merge-path algorithm for merging two sorted arrays on NVIDIA GPUs using CUDA. It includes a CPU baseline for comparison, performance benchmarking across various input sizes, and Python scripts for visualizing the results.

## Features

- **GPU-Accelerated Merge**: Efficient parallel merge using the merge-path partitioning technique.
- **CPU Baseline**: Reference implementation for correctness verification and performance comparison.
- **Benchmarking**: Automated timing and speedup measurements for different input sizes.
- **Visualization**: Python scripts to plot CPU time, GPU time, and speedup vs. input size.

## Prerequisites

- **CUDA Toolkit**: Version 10.0 or later. Ensure `nvcc` is in your PATH.
- **Python 3.x**: With `matplotlib` and `re` modules installed.
  - Install matplotlib: `pip install matplotlib`
- **NVIDIA GPU**: With CUDA support.

## How to Run

1. **Compile the CUDA Program**:
   ```
   make
   ```

2. **Run the Benchmark**:
   ```
   ./merge_path.exe
   ```
   This will:
   - Perform a small correctness test.
   - Run benchmarks for input sizes: 100,000; 500,000; 1,000,000; 5,000,000; and 10,000,000 elements.
   - Output results to `results.txt`.

3. **Visualize the Results**:
   ```
   python plot_results.py
   ```
   This will generate three plots:
   - CPU merge time vs. input size.
   - GPU merge time vs. input size.
   - Speedup (CPU/GPU) vs. input size.

## Project Structure

- `project.cu`: Main CUDA program containing the merge-path implementation, CPU baseline, and benchmarking logic.
- `plot_results.py`: Python script to read `results.txt` and generate performance plots.
- `results.txt`: Output file generated by the CUDA program with timing and correctness data.

## Algorithm Overview

The merge-path algorithm divides the merge task into independent subproblems that can be processed in parallel on the GPU. Each CUDA block handles a portion of the output array, using binary search to find the correct starting points in the input arrays.

## Performance Notes

- GPU timing measures only the kernel execution time using CUDA events.
- CPU timing includes the full merge operation.
- Speedup is calculated as CPU time divided by GPU time.
- Results may vary based on GPU architecture and input data distribution.
